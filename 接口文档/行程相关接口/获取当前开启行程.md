# 获取当前开启行程

## 接口信息

- **接口地址**: `/trip/enabled`
- **请求方法**: `GET`
- **认证方式**: JWT Token（在 Header 中携带）

## 请求参数

无请求参数（从JWT Token中获取用户ID）

## 响应参数

| 参数名 | 类型 | 说明 |
|--------|------|------|
| code | number | HTTP状态码，200表示成功 |
| message | string | 响应信息 |
| data | array | 开启行程列表数组 |
| data[].uuid | string | 行程唯一标识 |
| data[].title | string | 行程名称 |
| data[].text | string | 行程备注 |
| data[].startDate | string | 出发日期 |
| data[].endDate | string | 结束日期 |
| data[].cover | string | 行程封面图URL |
| data[].routePlan | array | 路线规划 |
| data[].isEnabled | boolean | 行程开启标记 |
| data[].createTime | string | 创建时间 |
| data[].updateTime | string | 更新时间 |
| timestamp | string | 响应时间戳 |

```json
  "timestamp": "2026-04-08T14:30:00.000Z"

```

## 业务逻辑

1. 用户必须登录才能查看开启行程
2. 系统自动从JWT Token中获取当前用户ID
3. 仅返回当前用户创建的且 isEnabled 为 true 的行程
4. 已删除的行程（is_deleted = 1）不会返回
5. 开启行程按更新时间倒序排列（最近更新的在前）

## 备注

- 接口地址使用 `/trip/enabled`
- 一个用户可以同时有多个开启的行程
- 如果用户还没有开启的行程，返回空数组 `[]`
