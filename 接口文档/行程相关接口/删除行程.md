# 删除行程

## 接口信息

- **接口地址**: `/trip/delete/:uuid`
- **请求方法**: `DELETE`
- **认证方式**: JWT Token（在 Header 中携带）

## 路径参数

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| uuid | string | 是 | 行程的唯一标识UUID |

## 响应参数

| 参数名 | 类型 | 说明 |
|--------|------|------|
| code | number | HTTP状态码，200表示成功 |
| message | string | 响应信息 |
| timestamp | string | 响应时间戳 |

### 成功响应示例

```json
{
  "code": 200,
  "message": "行程删除成功",
  "timestamp": "2026-04-08T15:00:00.000Z"
}
```

### 错误响应示例

**401 错误 - 未授权**

```json
{
  "code": 401,
  "message": "未登录或Token已过期",
  "timestamp": "2026-04-08T14:30:00.000Z"
}
```

**404 错误 - 行程不存在**

```json
{
  "code": 404,
  "message": "行程不存在或已被删除",
  "timestamp": "2026-04-08T14:30:00.000Z"
}
```

**403 错误 - 无权删除**

```json
{
  "code": 403,
  "message": "无权删除该行程",
  "timestamp": "2026-04-08T14:30:00.000Z"
}
```

## 业务逻辑

1. 用户必须登录才能删除行程
2. 通过URL路径参数传入行程的UUID
3. 系统检查行程是否存在且未被删除
4. 系统检查当前用户是否有权删除该行程（行程必须属于当前用户）
5. 使用软删除方式，将 is_deleted 字段设置为 1
6. 物理数据不会被删除，仅标记为已删除
7. 如果行程不存在或已被删除，返回404错误
8. 如果行程不属于当前用户，返回403错误

## 备注

- 接口地址使用 `/trip/delete/:uuid`
- UUID通过URL路径参数传入
- 采用软删除机制，数据不会从数据库中物理删除
- 已删除的行程无法再次删除
- 删除后，行程将不会在列表中显示，但数据仍然存在于数据库中
