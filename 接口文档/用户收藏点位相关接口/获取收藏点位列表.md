# 获取收藏点位列表

## 接口信息

- **接口地址**: `/user-collect/list`
- **请求方法**: `GET`
- **认证方式**: JWT Token（在 Header 中携带）
- **Content-Type**: `application/json`

## 请求参数

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| keyword | string | 否 | 搜索关键词，用于模糊匹配地点名称、详细地址、备注信息 |

### 请求示例

**不传关键词（查询全部）**：
```
GET /user-collect/list
```

**传入关键词（模糊搜索）**：
```
GET /user-collect/list?keyword=栈桥
GET /user-collect/list?keyword=青岛
GET /user-collect/list?keyword=打卡
```

### 搜索规则

- 关键词可匹配地点名称（title）、详细地址（address）、备注信息（remark）
- 支持模糊匹配，无需精确输入
- 关键词前后会自动添加通配符 `%`，实现包含匹配
- 关键词支持前后空格自动去除
- 不传关键词或传空字符串时，返回全部收藏点位

## 响应参数

| 参数名 | 类型 | 说明 |
|--------|------|------|
| code | number | HTTP状态码，200表示成功 |
| message | string | 响应信息 |
| data | array | 返回数据数组 |
| data[].uuid | string | 收藏点位唯一标识 |
| data[].title | string | 地点名称 |
| data[].latitude | number | 纬度 |
| data[].longitude | number | 经度 |
| data[].address | string | 详细地址 |
| data[].remark | string | 备注信息 |
| data[].createTime | string | 创建时间 |
| data[].updateTime | string | 更新时间 |

## 业务逻辑

1. 用户必须登录才能获取收藏点位列表
2. 系统自动获取当前登录用户的收藏点位
3. 支持关键词模糊搜索，同时匹配地点名称、详细地址、备注信息
4. 搜索条件使用 OR 连接，匹配任一字段即返回
5. 只返回未删除的收藏点位（is_deleted = 0）
6. 返回结果按创建时间倒序排列（最新的在前）

## 备注

- 接口地址遵循非RESTful风格，使用 `/user-collect/list`
- 返回数据不包含 id 字段，只包含 uuid 字段（对外使用 UUID）
