# 密码修改接口

## 基本信息

- **接口地址**: `PUT /user/password`
- **功能描述**: 修改当前登录用户的密码
- **认证方式**: JWT认证

## 请求参数

### 请求头

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| Authorization | string | 是 | Bearer \<token\> |
| Content-Type | string | 是 | application/json |

### 请求体 (JSON)

| 参数名 | 类型 | 必填 | 说明 | 验证规则 |
|--------|------|------|------|----------|
| oldPassword | string | 是 | 旧密码 | 6-20位字符 |
| newPassword | string | 是 | 新密码 | 6-20位字符 |

### 请求示例

```json
{
  "oldPassword": "OldPassword123",
  "newPassword": "NewPassword123"
}
```

## 响应参数

### 成功响应

| 参数名 | 类型 | 说明 |
|--------|------|------|
| code | number | 状态码，固定为 200 |
| message | string | 响应消息 |

### 成功响应示例

```json
{
  "code": 200,
  "message": "密码修改成功"
}
```

### 错误响应

#### 旧密码错误

```json
{
  "code": 400,
  "message": "旧密码错误",
  "timestamp": "2024-01-15T10:30:00.000Z"
}
```

#### 参数验证失败

```json
{
  "code": 400,
  "message": ["newPassword must be a string"],
  "timestamp": "2024-01-15T10:30:00.000Z"
}
```

#### 未授权（token无效或过期）

```json
{
  "code": 401,
  "message": "Unauthorized",
  "timestamp": "2024-01-15T10:30:00.000Z"
}
```

## 处理流程

1. **Token验证**: 验证请求头中的JWT token
2. **参数验证**: 验证旧密码和新密码格式
3. **用户查询**: 根据token中的userId查询数据库
4. **旧密码验证**: 使用bcryptjs比对旧密码是否正确
5. **密码加密**: 使用bcryptjs加密新密码
6. **数据更新**: 更新用户的密码
7. **返回结果**: 返回成功消息

## 密码加密说明

- **加密算法**: bcryptjs
- **加密强度**: 10 轮盐生成
- **存储方式**: 数据库中存储加密后的密码原文
