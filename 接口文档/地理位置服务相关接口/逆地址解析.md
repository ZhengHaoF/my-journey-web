# 逆地址解析

## 接口信息

- **接口地址**: `/proxy/geocoder`
- **请求方法**: `GET`
- **认证方式**: JWT Token（在 Header 中携带）
- **Content-Type**: `application/json`

## 功能说明

将经纬度坐标转换为文字地址及相关位置信息。该接口通过调用腾讯地图逆地址解析服务实现，广泛应用于物流、出行、O2O、社交等场景。

## 请求参数

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| latitude | number | 是 | 纬度（GCJ02坐标系），范围：-90 ~ 90 |
| longitude | number | 是 | 经度（GCJ02坐标系），范围：-180 ~ 180 |

### 请求示例

```
GET /proxy/geocoder?latitude=39.984154&longitude=116.30749
Authorization: Bearer <your_jwt_token>
```

## 响应参数

| 参数名 | 类型 | 说明 |
|--------|------|------|
| code | number | HTTP状态码，200表示成功，其他为腾讯地图返回的状态码 |
| status | number | 腾讯地图API状态码，0为成功，其他为异常 |
| message | string | 状态说明信息 |
| request_id | string | 本次请求的唯一标识 |
| result | object | 逆地址解析结果 |
| result.address | string | 标准格式化地址（行政区划+道路+门牌号） |
| result.formatted_addresses | object | 结合知名地点形成的描述性地址 |
| result.formatted_addresses.recommend | string | 推荐使用的地址描述 |
| result.formatted_addresses.rough | string | 粗略位置描述 |
| result.formatted_addresses.standard_address | string | 基于附近关键地点的精确地址 |
| result.address_component | object | 地址部件 |
| result.address_component.nation | string | 国家 |
| result.address_component.province | string | 省/直辖市 |
| result.address_component.city | string | 市/地级区 |
| result.address_component.district | string | 区/县级市 |
| result.address_component.street | string | 道路 |
| result.address_component.street_number | string | 门牌号 |
| result.ad_info | object | 行政区划信息 |
| result.ad_info.nation_code | string | 国家代码（ISO3166标准3位数字码） |
| result.ad_info.adcode | string | 行政区划代码 |
| result.ad_info.city_code | string | 城市代码 |
| result.ad_info.name | string | 行政区划名称 |
| result.ad_info.location | object | 行政区划中心点坐标 |

### 错误响应示例

**400 错误 - 请求参数错误或腾讯地图API错误**

```json
{
  "code": 400,
  "status": 310,
  "message": "key格式错误",
  "request_id": "xxx-xxx-xxx"
}
```

**401 错误 - 未授权**

```json
{
  "code": 401,
  "message": "未登录或Token已过期",
  "timestamp": "2026-04-09T10:00:00.000Z"
}
```

**503 错误 - 腾讯地图服务不可用**

```json
{
  "code": 503,
  "message": "无法连接到腾讯地图服务，请稍后重试",
  "timestamp": "2026-04-09T10:00:00.000Z"
}
```

## 业务逻辑

1. 用户必须登录才能使用逆地址解析服务
2. 前端传入经纬度（GCJ02坐标系），后端携带 API Key 调用腾讯地图服务
3. API Key 存储在环境变量中，不会暴露给前端
4. 判断腾讯地图返回的 status 状态：
   - status = 0：解析成功，添加 code = 200
   - status ≠ 0：解析失败，原样返回 status、message 等信息
5. 异常处理：
   - 网络连接失败：返回 503 错误
   - 腾讯地图API错误：返回 400 错误

## 技术实现

- **HTTP 客户端**: axios
- **超时设置**: 10秒
- **API 地址**: https://apis.map.qq.com/ws/geocoder/v1/
- **坐标系**: GCJ02（腾讯、高德、微信使用）

## 备注

- 坐标系说明：国内主要使用 GCJ02（火星坐标系），腾讯地图、高德地图、微信位置等均使用此坐标系
- 坐标系转换：如果前端使用 WGS84（GPS原始坐标），需要转换为 GCJ02 才能使用此接口
- API Key 保护：API Key 存储在后端环境变量中，前端无法直接访问
- 错误处理：所有异常使用 HttpException 抛出，由全局异常过滤器统一处理
